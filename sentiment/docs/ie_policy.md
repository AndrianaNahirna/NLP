# IE Policy

Цей документ описує специфікацію витягу для трьох типів полів (DATE, AMOUNT, LOCATION), які присутні у корпусі відгуків.

## 1. DATE (Дати)
* **Що саме вважаємо полем:** Календарні дати, які вказують на час покупки, доставки або створення замовлення. Розпізнаються числові формати (DD.MM.YYYY, DD.MM.YY) та текстові формати з назвами місяців українською мовою.
* **Який формат на виході (нормалізація):** `YYYY-MM-DD`.
* **Що НЕ вважаємо полем (негативні приклади):** Версії програмного забезпечення (наприклад, "оновився до 2.0.1"), номери моделей (наприклад, "модель 10.12"), будь-які числові послідовності з крапками, які не утворюють валідну дату.
* **5 типових edge cases  з датасету:**
    1. Відсутність століття у форматі року ("12.04.16" нормалізується до "2016-04-12").
    2. Невалідна кількість днів або місяців (наприклад, "32.13.2023" не повинно розпізнаватися як дата).
    3. Нульові значення у днях (наприклад, "00.05.2019" повинно ігноруватися).
    4. Дні та місяці, написані без нулів на початку (наприклад, "1.2.2020").
    5. Рядки, що імітують дату, але є версією ПЗ ("версія 2.0.12" викликає False Positive без належних обмежень).

## 2. AMOUNT (Грошові суми)
* **Що саме вважаємо полем:** Грошові витрати, ціни на товари або суми повернень. Визначаються як числа (цілі або десяткові дроби), після яких (або поруч з якими) стоїть ідентифікатор валюти зі словника (грн, гр, ₴, $, usd, uah).
* **Який формат на виході (нормалізація):** `value (float) + currency (UAH/USD/UNKNOWN)`. Наприклад, "499.0 UAH".
* **Що НЕ вважаємо полем (негативні приклади):** Відсотки ("ціна впала на 100%"), кількість товарів ("замовив 2 шт"), технічні характеристики або об'єми пам'яті ("64 Гб", "важить 2 кг"), числа без вказівки валюти ("заплатив 500").
* **5 типових edge cases з датасету:**
    1. Використання коми замість крапки як розділювача для копійок ("1,5 грн" нормалізується до "1.5 UAH").
    2. Скорочення назв валют з помилками або без крапок ("79 гр" замість "грн").
    3. Використання символів валют замість слів ("100$", "3000 ₴").
    4. Великі числа без пробілів для розрядів ("21841 грн").
    5. Використання англомовних абревіатур в україномовному тексті ("50 usd", "1500 uah").

## 3. LOCATION (Міста)
* **Що саме вважаємо полем:** Назви українських міст, куди доставлявся товар або де була здійснена покупка. Визначаються за словником найпопулярніших міст та їх відмінків.
* **Який формат на виході (нормалізація):** Базова форма міста у Називному відмінку з великої літери (наприклад, "Київ", "Львів").
* **Що НЕ вважаємо полем (негативні приклади):** Назви областей ("Львівська область", "Київщина"), прикметники, утворені від назв міст ("київський торт", "одеський гумор").
* **5 типових edge cases з датасету:**
    1. Написання назви міста з малої літери у відгуках ("зустріч у києві").
    2. Зміна кореня або закінчення слова при відмінюванні ("у Львові", "до Києва").
    3. Однокореневі слова, що є назвами брендів (наприклад, "Київстар" не повинно спрацьовувати як "Київ").
    4. Назви міст, які можуть використовуватися в іншому значенні (наприклад, "Дніпро" як місто та як річка/назва техніки).
    5. Прийменники, приклеєні до назви через відсутність пробілу ("вКиєві"), що вимагає правильної токенізації.
