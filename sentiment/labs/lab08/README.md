# Lab 8: Topic Modeling

**1. Який корпус аналізували**
Аналізувався корпус відгуків клієнтів про українські інтернет-магазини електроніки (Rozetka, Citrus). Після попередньої очистки, лематизації та фільтрації (видалення занадто коротких та порожніх текстів) розмір корпусу склав 3034 документи.

**2. Які моделі запускали**
Було протестовано дві класичні моделі тематичного моделювання:
* LSA (Latent Semantic Analysis)
* LDA (Latent Dirichlet Allocation)

**3. Які k перевіряли**
Для обох алгоритмів експерименти проводилися з кількістю тем k = 5, 8 та 10.

**4. Які теми вийшли найкращими**
Найкращі результати для інтерпретації показала модель LDA (зокрема при k=10). Вдалося виділити дуже точні бізнес-проблеми:
* **Недокомплектація товару** (скарги на відсутність деталей та кріплень при доставці).
* **Гарантійний ремонт техніки** (поломки складної електроніки, здача в сервісний центр).
* **Затримки з поверненням коштів** (тривале очікування фінансів клієнтами після повернення товару).

**5. Які теми вийшли поганими**
* **LSA (k=5), Теми 3 та 4 (Duplicate topics):** Модель не змогла розділити сюжети і видала дві ідентичні теми, що підтверджено однаковими топ-документами в обох кластерах.
* **LDA (k=10), Тема 4 (Domain noise / Mixed topic):** Тема згрупувала випадкові дати оформлення замовлень (технічний шум) та змішала абсолютно різні категорії товарів (навігатори, грилі), не утворивши логічного сюжету.

**6. Що показало порівняння LSA vs LDA**
Порівняння виявило беззаперечну перевагу **LDA** для даного корпусу. LSA була схильною до дублювання тем та створення розмитих кластерів, побудованих навколо загальновживаних слів («магазин», «телефон»). LDA натомість змогла виділити специфічні клієнтські болі. Це підтвердилося і розрахунком метрики Coherence (C_v): найкращий результат LDA склав 0.5128 (при k=8), тоді як максимум LSA — лише 0.4721.

**7. Чи topic modeling взагалі корисний для вашого корпусу**
Так, тематичне моделювання виявилося надзвичайно корисним інструментом аналітики. Якщо звичайний аналіз тональності дозволяє лише зрозуміти, чи задоволений клієнт загалом, то Topic Modeling автоматично розбив тисячі відгуків на конкретні бізнес-процеси. Це дозволило побачити, що негатив генерується не абстрактно, а через цілком конкретні недоліки сервісу: тривалі процедури повернення коштів, відсутність комплектуючих та погану роботу сервісних центрів. Такі інсайти можна напряму використовувати для покращення роботи бізнесу.
