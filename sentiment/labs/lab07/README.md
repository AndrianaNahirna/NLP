# Lab 7: Linear SVM, Char-ngrams & PR-Thresholds

**1. Яка підзадача класифікації:**
Бінарна класифікація тональності україномовних відгуків (positive / negative).

**2. Який baseline із ЛР6 взято для порівняння:**
Logistic Regression + TF-IDF (word, 1-2) з параметром `class_weight="balanced"` (Macro-F1 = 0.8407).

**3. Які SVM-варіанти перевірені:**
* LinearSVC на словах (TF-IDF `word`, 1-2) без балансування та з `class_weight="balanced"`.
* LinearSVC на символах (TF-IDF `char_wb`, 3-5) без балансування та з `class_weight="balanced"`.

**4. Чи був imbalance:**
Так, у датасеті є стабільний дисбаланс класів на всіх сплітах: ~63% negative проти ~37% positive.

**5. Який поріг обрано і чому:**
* **Оптимальний (0.000):** Стандартний поріг виявився математично найкращим для максимізації загального F1-score, оскільки балансування ваг вже зсунуло гіперплощину.
* **Кастомний (-0.500, Recall-first логіка):** Був обраний під гіпотетичну бізнес-задачу "не пропустити жодного позитивного відгуку" (наприклад, для видачі бонусів клієнтам). Зниження порогу дозволило різко підняти Recall класу positive з 0.70 до 0.92, хоча й ціною падіння Precision.

**6. Яка модель виявилась найкращою:**
Серед нових моделей найкращою стала **LinearSVC (word + balanced)** з Macro-F1 = 0.8262. Використання `char-ngrams` результати лише погіршило. Однак глобальним лідером залишився простіший baseline з ЛР6 (Logistic Regression), що доводить: складніший підхід не завжди гарантує кращий результат.

**7. Що робити далі:**
* Очистити датасет від хибної розмітки (noisy labels).
* Вирішити проблему змішаної тональності (overlap класів), можливо, додавши третій "нейтральний" клас або перейшовши до multi-label формату.
* Спробувати моделі на базі щільних векторних представлень (наприклад, Word2Vec, FastText або Transformer-моделі), оскільки TF-IDF лінійні моделі погано вловлюють сарказм та заперечення (напр., "не є підробкою").
