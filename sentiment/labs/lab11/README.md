# Лабораторна робота №11: LLM Extraction Pipeline

Ця робота присвячена побудові надійного інженерного пайплайну для витягування структурованих даних з неструктурованого тексту за допомогою LLM (Llama-3-8B) з використанням підходу **schema-first**.

## 1. Екстракція-кейс
Обрано задачу аналізу відгуків користувачів на товари та послуги в українському сегменті e-commerce. Мета — витягти ключові атрибути для подальшої автоматизованої аналітики.

## 2. JSON Schema
Використовується сувора схема, що включає 5 полів:
* `sentiment_type`: (enum) positive, negative, neutral.
* `mentioned_aspects`: (array) список згаданих характеристик.
* `advantages`: (string/null) опис переваг.
* `disadvantages`: (string/null) опис недоліків.
* `rating_mentioned`: (number/null) числова оцінка.

## 3. Baseline Prompt
Базовий промпт містить інструкцію рольової моделі ("Ти — помічник..."), формальний опис JSON-схеми, Few-shot приклад для демонстрації формату та безпосередньо текст відгуку, обмежений трьома лапками.

## 4. Валідатор
Реалізовано дворівневий валідатор:
1. **JSON Parse Validation**: перевірка синтаксичної коректності рядка за допомогою бібліотеки `json`.
2. **Schema Validation**: перевірка відповідності структури та типів даних за допомогою бібліотеки `jsonschema`.

## 5. Repair Loop
Алгоритм автоматичного виправлення працює за логікою:
* Якщо валідація не пройдена, формується `repair_prompt`.
* У промпт передається: оригінальний текст, зламаний вихід та повідомлення про конкретну помилку від валідатора.
* Максимальна кількість спроб виправлення: 2.

## 6. Метрики (Valid JSON Rate)
За результатами тестування на 25 прикладах:
* **Raw Valid JSON Rate (до repair):** 68.00%
* **Post-Repair Valid JSON Rate (після repair):** 100.00%
* **Ефективність:** Repair Loop успішно врятував 100% проблемних випадків.

## 7. Залишкові проблеми
Основною проблемою залишаються помилки типізації (`Schema Validation Error`). Локальна модель схильна повертати масив рядків для поля `disadvantages`, коли недоліків декілька, хоча схема вимагає об'єднаний рядок (string). Також спостерігаються поодинокі семантичні помилки у визначенні тональності в неоднозначних контекстах.
